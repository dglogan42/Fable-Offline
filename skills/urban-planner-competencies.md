# Urban planner competencies

**WHEN_TO_USE:** Career or study questions about urban planning / urban design; skill audits for planners; learning plans; portfolio advice; linking postgraduate webinars (e.g. Master of Urban Design) to practice competencies; or designing an agent that assists planning work (GIS notes, policy maps, stakeholder plans) without replacing licensed professionals.

## Stance
Urban planning blends **spatial analysis**, **policy knowledge**, and **stakeholder management** to design and implement community land-use strategies. Technical tools matter; so do negotiation, empathy, and creative problem-solving. This skill **coaches competence frameworks** — it does not practice law, certify planners, or approve developments.

**Not legal, planning, or engineering advice.** Zoning and consent decisions require local professionals and statutory processes.

---

## Four competence areas

### 1. Technical & analytical
| Competency | What good looks like |
|------------|----------------------|
| **GIS** | Map infrastructure, analyse land use, visualise spatial layers; cite CRS/data sources |
| **Data analysis & statistics** | Interpret demographic, economic, environmental data; forecast needs with uncertainty labels |
| **CAD & 3D modelling** | Produce spatial plans / layouts others can critique; version assumptions |
| **Regulatory knowledge** | Local zoning, building codes, environmental rules, resource management — **jurisdiction-specific**; never invent bylaws |

### 2. Communication & interpersonal
| Competency | What good looks like |
|------------|----------------------|
| **Stakeholder engagement** | Involve residents, business, iwi/mana whenua (where relevant), environment groups with clear methods |
| **Public speaking & presentation** | Translate technical work for boards and the public without hiding trade-offs |
| **Mediation & negotiation** | Facilitate when developers, policymakers, and residents conflict |
| **Team collaboration** | Work with architects, civil engineers, economists, transport planners |

### 3. Design & strategic
| Competency | What good looks like |
|------------|----------------------|
| **Creative vision** | See alternative social/physical futures for a place; test against constraints |
| **Master planning & revitalisation** | Shape public space, density, open space, sustainable transit systems |
| **Project evaluation** | Feasibility, sustainability, equity/social impact — with criteria and evidence |

### 4. Management & organisation
| Competency | What good looks like |
|------------|----------------------|
| **Decision making** | Weigh options under budget and land constraints; document why |
| **Leadership & organisation** | Multi-project delivery; staff oversight; alignment with mandates and plans |

## audit-competencies

Score each row **1–5** (1 = no evidence, 5 = can teach others) with **evidence** (project, course, work sample) or **gap**.

Output:
1. Verdict first — strength cluster + biggest gap cluster  
2. Score table (four areas, all rows)  
3. Top 3 gaps with one concrete practice task each  
4. What not to claim (e.g. “I know all zoning” without jurisdiction)  

---

## plan-growth

**Input:** current level, target role (council, consultancy, NGO, research), time horizon.

**Steps:**
1. Pick **one** technical skill and **one** interpersonal skill per quarter (don’t boil ocean).  
2. Map to real artifacts: map package, policy memo, engagement plan, 3D option set.  
3. Prefer local statutory literacy (district plan / RMA or local equivalent) over generic “urban theory only.”  
4. End with **this month’s build** (one deliverable).  

Align with agentic habits: goal → do → verify with a critic (peer/mentor) → stop or iterate (`loop-engineer`).

---

## map-programme

When evaluating a course, webinar, or degree (e.g. Master of Urban Design):

1. Map claimed learning outcomes → the four competence areas.  
2. Flag marketing vs evidence (use `education-claim-audit` if accreditation claims appear).  
3. List what the programme **won’t** replace (years of statutory practice, GIS portfolio).  
4. Next verification: faculty, studio load, software stack, studio city/context.  

Cross-link: `knowledge/privacy/uoa-eloqua-pg-webinar-hosts.md` if the surface is a UoA marketing LP (privacy ≠ curriculum quality).

## design-planning-agent

Design an agent that **assists** planners (not an autonomous consent authority).

| Component | Rule |
|-----------|------|
| Goal | Analyse, summarise, draft options, structure engagement — **HITL on recommendations** |
| Tools | Read local plan excerpts user provides; GIS **data** user supplies; write memos under workspace |
| Forbidden | Inventing zone rules; fabricating flood/hazard layers; submitting statutory applications |
| Verifier | Re-derive numbers; label guesses; cite user-supplied sources only |
| Privacy | Stakeholder contact lists and submissions = PII → use `privacy-design-planner` |

Stack with Fable: `/team` for research→write→critic on policy memos; `/engineer` for scored option evaluation; `/privacy` if web tooling is involved.

## plan-engagement

Structure a stakeholder process:
1. Purpose and decision rights (who decides what)  
2. Who is affected (map interests, power, information needs)  
3. Methods (workshop, survey, drop-in, hui) + accessibility  
4. What is negotiable vs fixed (budget, hazard, legal)  
5. Feedback loop (how input changes the plan)  
6. Risk: tokenism — require at least one “what changed because of engagement” field  

---

## Output shape (default coaching)

1. **Verdict first** (fit / gap / next step)  
2. Competence area focus (which of the four)  
3. Concrete practice task (≤ 2 weeks)  
4. Artifact to produce (file or map package name)  
5. Verify how (mentor, checklist, public criteria)  
6. Explicit non-claims  

---

## Forbidden
- Approving land use or inventing bylaws  
- Presenting as a licensed planner without credentials  
- Hiding equity or environmental trade-offs to “sell” a vision  
- Treating webinar registration marketing as proof of competence
