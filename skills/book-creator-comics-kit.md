## comic-workflow

Typical student path (UI labels VERIFY LIVE in app):

1. Create / open book → choose comic-friendly page setup  
2. **Add panels** for story beats (wide establishing → action → reaction)  
3. **Photos / camera** into panels (or draw/import assets school allows)  
4. **Stickers** for characters/props if using library  
5. **Speech bubbles + captions** for dialogue and narration  
6. Apply **comic styles / effects** (shadows, borders, onomatopoeia stickers: Bang, Zoom, etc.)  
7. Optional audio/read-aloud features if product offers (VERIFY LIVE full features list)  
8. Peer review → revise → **publish/share** per teacher rules

## panel-recipe

| Goal | Layout idea (from marketing panel set) |
|------|----------------------------------------|
| Simple A/B | Double grid |
| Rising action | 3-panel horizontal or diagonal |
| Complex beat | 4-panel grid |
| Focus beat | Single large panel + caption |

Rule of thumb: **one clear action per panel**; leave bubble space.

---

## bubble-style-recipe

1. Speech = spoken dialogue; thought = internal (product thought-bubble options)  
2. Captions = narrator / time/place  
3. Funky text / stickers for sound effects (Crash!, Zoom!) — use sparingly  
4. Keep text readable (contrast, size); accessibility first
