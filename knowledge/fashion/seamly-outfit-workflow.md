## What Seamly is

| Fact | Detail |
|------|--------|
| Category | Free **open-source apparel CAD** (pattern making) |
| Also known as | Seamly2D (community/product naming) |
| Platforms | Windows, Linux, macOS (per vendor) |
| Value prop (vendor) | Formula-based sizing, multi-size from one pattern, reduce walk-off seams / rework |
| Not | Auto Instagram poster, 3D virtual try-on mandatory, or body-shame tool |

Optional vendor integrations (e.g. body scan → measurements) may appear on seamly.io — treat as **optional**, re-check current docs.

## Download path (user CLICK)

1. Open **[seamly.io/download](https://seamly.io/download/)**  
2. Submit first name, last name, email (vendor form)  
3. Receive email with download link  
4. Install for your OS; launch app  

Privacy: vendor states commitment to protect personal information and not sell/share for unrelated reasons — read [Privacy Policy](https://seamly.io/privacy-policy) on their site.

Fable never stores the download form contents in git.

## Outfit loop (select ↔ create)

```
Occasion + vibe
    → select-outfit (closet)
    → or create-outfit-brief (new look)
        → measure-sheet (if sewing)
        → seamly-project-plan (if drafting)
        → materials-list + sew / muslin
        → fit-iteration
        → hand-off-slay (instagram-selfie-selector)
```

## Seamly project phases (high level)

| Phase | User does in Seamly | Fable helps with |
|-------|---------------------|------------------|
| 1 Measurements | Enter chart / formulas | measure-sheet checklist |
| 2 Draft | Points, curves, pieces | Garment type + construction order |
| 3 Sizes | Multi-size formulas | Which sizes to generate (user list) |
| 4 Layout | Grain, seam allowance | Notions / fabric width questions |
| 5 Cut & sew | Outside CAD | materials-list, fit-iteration |
| 6 Show | Photo | Instagram skill |

## File hygiene (local)

| Keep | Avoid in public git |
|------|---------------------|
| Curated markdown notes | Full body photos, measurement dumps with names |
| Pattern **ideas** | Commercial pattern PDFs you don’t have rights to share |
| `knowledge/fashion/_local/` | Anything with real client measurements |

Typical Seamly-related extensions (do not commit client work): project/pattern files as produced by the app — keep under `_local/` or outside repo.
